## 第 2 页 · 痛点：刷过 ≠ 学会

**核心数据 / 金句**：
- 一句话痛点："90% 的人刷完一条科普视频，第二天只记得 UP 主的语气，不记得知识本身。"

**三方困境**（三栏布局）：

| 用户 | 平台 | 创作者 |
|---|---|---|
| "看过"不等于"学会"，没有追问接口 | 优质长视频因低完播率被算法折叠 | 深度努力被"刷过"而非被"读懂" |

**视觉建议**：左中右三个 icon + 短句，配色用灰冷调强化"痛"感。

## 第 6 页 · 工程亮点（评审会问的点）

1. **Schema 强制结构化**：`response_schema` + Pydantic 双层校验，从源头杜绝 JSON 解析失败
2. **图谱结构约束**：Prompt 强制 `[核心]/[分支]/[细节]` 分层 + 关系标注，无孤立节点
3. **优雅降级**：Pro 503 自动切 Flash，demo 现场不翻车
4. **真实进度流**：SSE 四阶段反映 Gemini File API 真实状态，不是假动画
5. **零延迟 demo 通道**：预缓存案例绕过实时上传链路，评审秒看效果
6. **会话恢复**：缓存过期后分析数据持久化，对话连续性不断
7. **多平台反爬**：yt-dlp + 桌面 UA spoof + cookie + 重试，抖音/B 站/YouTube 通吃

## 第 7 页 · 商业价值 + 社会价值

**商业价值**（左半）：
- **C 端订阅**：抖音 Pro 用户专享"科普解构"功能
- **B2B 教育**：K12 / 高校 / 职业教育内容方接入
- **企业内训**：把内训视频自动拆成课程包

**社会价值**（右半）：
- **教育公平**：让优质科普视频"被读懂"，降低学习门槛
- **终身学习**：碎片视频沉淀为可复用的知识图谱
- **创作者激励**：深度内容获得"被学会"的衡量指标，而非完播率
- **优质内容全球传播**：结构化产物天然支持翻译与跨语言学习
