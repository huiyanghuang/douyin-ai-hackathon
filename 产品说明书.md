# 科普解构器（Sci-Decoder）

### 让视频变成你的私人课程

> 抖音黑客松 2025 · 赛道二「抖音精选-内容重构：让视频成为你的生活搭子」  
> 团队 · 25 小时全栈交付 · 线上 Demo：<https://douyinhackathon.xinguangtreehole.com>

---

## 一句话定位

**上传一段科普视频，AI 把它拆成「概念图谱 + 知识卡片 + 追问对话 + 知识测验」四件套——从"看过"到"学会"的最短路径。**

---

## 一、核心痛点洞察

### 1.1 用户侧：科普视频的"高浏览，低留存"悖论

抖音精选上每天有海量优质科普视频被生产、被推送、被点赞，但用户的真实体验是：

| 表面行为 | 真实状态 |
|---|---|
| 看完点赞收藏 | 三天后只记得 UP 主语气，记不住三个核心知识点 |
| 评论区"学到了" | 没有任何可调用的能力沉淀 |
| 算法不停推同主题视频 | 不同视频的观点冲突、知识盲区无人为你梳理 |
| "看完想再深问一句" | 没有任何接口可以追问，只能换关键词重搜 |
| "想测验自己是否懂了" | 没有任何形式的及时反馈 |

**根本矛盾**：**视频天然是"线性、单向、留不住痕迹"的媒介**，而**学习的本质是"结构化、双向、可被复用"**。两者错位带来巨大的认知漏斗损耗。

### 1.2 平台侧：算法红利触顶，"留存深度"才是下一程

抖音精选已经把"找到你感兴趣的内容"做到极致，但"让你真的学到东西"这一层还几乎是空白。用户停留时长会因算法疲劳而衰减；只有**真正产生认知收益的内容消费**，才能续命留存。

### 1.3 创作者侧：优质长视频被算法折叠

3 分钟以上的深度科普视频，因完播率较低而被算法降权。但这些恰恰是**信息密度最高、最值得被结构化的内容**。我们提供的工具，让创作者的努力被"读懂"而不是"刷过"。

### 1.4 一句话总结痛点

> **当下的视频消费是"刷"出来的多巴胺，不是"看会了"的获得感。我们要把"看过"变成"看懂"，再变成"会用"。**

---

## 二、我们的解决方案

将一段 ≤10 分钟的科普视频自动拆解为**四个相互联动的学习产物**：

| 产物 | 形态 | 解决的痛点 |
|---|---|---|
| **知识卡片** | 概念名 + 类比解释 + 时间戳 | 把"听懂"压缩到 30 秒 |
| **概念图谱** | D3 力导向图，节点分层、关系标注 | 把碎片串成可视化的知识网络 |
| **AI 追问** | 基于视频结果的 grounded 对话 | 解决"看完想再问一句" |
| **知识测验** | 三认知层级的即时反馈题目 | 把"我以为我懂了"变成"我真的懂了" |

再叠加三层增强：

- **导出 .md 笔记**：一键沉淀到本地知识库
- **猜你想看**：AI 抽延伸关键词 → 一键跳转抖音/B 站站内搜索
- **历史回看 + 鉴权**：跨设备同步个人学习轨迹

### 关键设计原则

1. **单 LLM 调用拿到完整 JSON**：通过 `response_schema` 强制输出，避免"先 STT，再切片，再分别提取"的传统链路，省时间也省失败面。
2. **进度对真实流程不撒谎**：SSE 推送的四个阶段（uploading / analyzing / structuring / done）逐一映射 Gemini File API 的真实状态机，不是前端假动画。
3. **Pro → Flash 自动降级**：高峰期 503 不会让评审现场翻车。
4. **预缓存 demo 通道**：评审点击预置案例 → 直接返回缓存 JSON，零 LLM 调用，秒级响应。

## 四、多模态交互流程设计（UX）

### 4.1 输入侧：多模态收口

| 模态 | 入口 | 实现 |
|---|---|---|
| 文件（视频） | 中央拖拽框 + 点击选择 | 浏览器 File API，上传 MP4 ≤ 100MB |
| 链接（URL） | "解析链接"输入框 | 后端 yt-dlp 异步拉取 |
| 鼠标（拖拽） | dropzone + 进度条 | HTML5 DnD events |
| 键盘（追问） | 聊天输入框 | POST /api/chat |
| 视觉（图谱） | 拖动节点 / hover / 滚轮缩放 | D3 v7 force + zoom |

### 4.2 完整用户旅程

```
[1] 进入首页（hero + 历史 + demo 卡片）
        │
        │ 用户拖拽 MP4 / 粘贴 B 站链接 / 点击 demo 卡
        ▼
[2] 全屏进度遮罩（4 阶段图标 + 中文文案）
        │  上传 → 理解 → 构建 → 完成
        ▼
[3] 结果区淡入，左侧纵向 Tab（4 个）+ 右侧"猜你想看"边栏
        │
        ├─[Tab 1] 知识卡片：title + summary + 概念网格 + 时间戳侧栏
        ├─[Tab 2] 概念图谱：力布局/径向 mindmap 切换 + 一键讲解 tour
        ├─[Tab 3] AI 追问：建议问句 + markdown 渲染气泡
        └─[Tab 4] 知识测验：逐题反馈 + 总得分 + 重做
        │
        │ 用户选择延伸动作
        ▼
[4] 导出 .md 笔记 / 点击猜你想看跳站内 / 登录后同步历史
```

### 4.3 多模态输入 → 多模态输出的真实闭环

| 用户给出 | 系统给回 |
|---|---|
| 一段视频（视觉 + 听觉） | 文字（卡片）+ 图（D3 图谱）+ 交互（quiz）+ 对话（chat） |
| 一个点击（图谱节点） | 浮层解释 + 类比 + 时间戳跳转 |
| 一个问题（自然语言） | grounded markdown 回答 + 引用 key_points |
| 一个选择（quiz 选项） | 即时对错 + 详细解析 + 总分动画 |

**每一种输入都有对应的多模态输出，且形式之间互相联动**——卡片里的概念名在图谱里高亮，图谱节点在测验里出现，测验里的解析又能直接追问。这才是"多模态交互"的实质，而非简单的"既能上传又能输入文字"。

### 4.4 视觉与设计语言

- **主色**：深空蓝 `#1a1d29` 底 + 紫色 `#6366f1` 强调 + 白色文字
- **圆角**：12px 统一，避免锐角带来的紧迫感
- **字体**：系统默认 sans-serif，标题 700 字重
- **图谱配色**：柔和的科技蓝/紫渐变，避免花哨
- **微交互**：进度条平滑过渡、tab 切换淡入淡出、答对/答错的微动画

## 五、核心创新点

| # | 创新点 | 同类方案的痛点 | 我们的做法 |
|---|---|---|---|
| 1 | **单次 LLM 调用拿结构化产物** | 传统链路：STT → 切片 → 多次 LLM 提取，慢且失败面大 | Gemini 视频直读 + response_schema |
| 2 | **概念分层 + 连通图约束** | LLM 抽出来的概念常常孤立、扁平 | Prompt 里强制 `[核心]/[分支]/[细节]` 标签 + concept_relations 必须连通所有节点 |
| 3 | **量化动态规则** | 短视频和长视频用同一套节点数 | Prompt 里按时长分档（3/8 分钟）动态调节 key_concepts / key_points / quiz 数量 |
| 4 | **预缓存通道 + 实时通道并行** | 评审现场依赖实时上传容易翻车 | demos.json 离线生成 → 评审秒级体验，实时通道留给"真用户" |
| 5 | **图谱讲解 tour 动画** | 静态图谱看完不知道重点 | 一键播放：按"核心→分支→细节"顺序自动高亮 + 浮层讲解 |
| 6 | **历史回看降级** | 内存 task TTL 失效后追问就废了 | 前端把 analysis 整体随 chat 请求回传，永远能追问 |

## 六、商业价值与社会价值

### 6.1 商业价值

#### 短期（嵌入抖音精选生态）
- **付费学习模式订阅**：抖音精选 Pro 用户每月得到 X 条视频的深度解构额度，B 端定价 ¥19.9 / 月起。
- **创作者赋能工具**：知识创作者可在发视频时一键生成解构数据，附在视频下方，提升完播率与点赞率。
- **数据飞轮**：用户答题正确率、追问深度，反向给视频内容打"教学价值评分"，平台据此改进推荐质量。

#### 中期（B2B 输出能力）
- **K12 / 高校在线教育**：网课平台接入，把直播课与录播课实时结构化，做"AI 助教"。
- **企业内训**：把内部录播会议、培训视频转化为结构化知识库。
- **教辅出版**：教辅图书附 AR 二维码，扫码看视频→自动给出配套测验。

#### 长期（重构内容消费方式）
- 当"看视频"自带"学会工具"成为习惯，**短视频本身的产品定义会被扩展**。"刷视频"和"上一门微课"之间的边界消失，平台变成最大的**自适应学习系统**。

### 6.2 社会价值

| 维度 | 价值描述 |
|---|---|
| **教育公平** | 农村、老年、英语弱势群体能与一线城市学生消化同等深度的科普内容，因为 AI 帮你把高密度信息翻译到"初中生能懂"的层级 |
| **终身学习** | 把每一次刷视频都变成可累积的能力沉淀，对抗"算法茧房刷完即弃"的零和体验 |
| **知识沉淀** | 用 AI 把算法推送的流量沉淀成用户可控的个人知识库，让"被推送"变成"主动学" |
| **创作者激励** | 深度内容创作者的努力第一次有了**可被量化的认知收益证明**，平台据此可以反向回报优质内容生产者，对抗劣币驱逐良币 |
| **多语言扩展** | 同一套架构天然支持把中文视频结构化给非中文用户学习，未来可以让中国优秀科普内容走向全球 |

### 6.3 评审锚点对照

| 赛题评审锚点 | 我们的回应 |
|---|---|
| 场景与问题洞察（15%） | "高浏览低留存"+"主动学习无接口"+"创作者优质长视频被折叠"三层痛点 |
| AI 能力与产品结合（20%） | Gemini 视频直读直接产出结构化 JSON 是核心，不是装饰 |
| 体验完整性（15%） | 上传→四 Tab→追问→测验→导出→延伸，完整闭环已部署上线 |

| 创新性与延展潜力（20%） | 多模态输入 → 多模态输出 + 数据飞轮 → 平台级延伸 |

## 七、未来展望

- **实时直播解构**：直播流边播边结构化，弹幕区变成结构化提问区
- **多视频对比**：上传同主题 N 条视频 → 自动梳理观点差异、共识、互补
- **个性化重述**：根据用户教育背景与历史答题数据，把同一段视频翻译成不同认知层级的版本
- **AR / 语音入口**：手机 AR 镜头识别现实物体 → 自动调用关联视频结构化结果
- **创作者反向赋能**：把用户答错率高的题反推给创作者，作为内容改进信号
